# KYC Remediation & Risk Monitoring Dashboard

## Project Overview
This project analyses synthetic corporate-banking KYC records to assess portfolio health, document quality, and remediation priorities. The objective is to help a financial institution identify high-risk clients, surface documentation gaps, and convert KYC findings into a clear operational action plan that supports remediation and audit readiness. :contentReference[oaicite:0]{index=0}

## Tools Used
- **Power BI** — dashboarding and executive-level reporting  
- **DAX** — KPI calculations and segmentation logic  
- **Excel** — data preparation and quality checks :contentReference[oaicite:1]{index=1}

## Key Portfolio Metrics
- **1,000 clients reviewed**
- **16,807 documents tracked**
- **41 high-risk clients**
- **92.1% average completeness**
- **99 PEP-linked files**
- **593 watchlist files**
- **69 clients requiring remediation** :contentReference[oaicite:2]{index=2}

## Key Findings
- The overall portfolio appears broadly stable, but a smaller population of high-risk and PEP-linked files requires closer monitoring because those cases carry disproportionate regulatory exposure. :contentReference[oaicite:3]{index=3}
- The main operational bottleneck is document quality rather than risk scoring alone: **9,575 documents were verified, 5,709 were pending verification, 1,523 failed verification, and 10,545 were expired**. This indicates pressure on onboarding, refresh, and review workflows. :contentReference[oaicite:4]{index=4}
- Enhanced due diligence pressure is concentrated in **Art/Jewelry Dealers** and **Money Service Businesses**, suggesting tighter onboarding and review controls are needed in those sectors. :contentReference[oaicite:5]{index=5}
- Of the **69 remediation files**, **15 medium-risk cases** should be prioritised first, while **54 low-risk cases** can be cleared in controlled batches. :contentReference[oaicite:6]{index=6}

## Business Value
This dashboard converts raw KYC data into an operational decision tool. It helps compliance teams:
- identify which files require immediate attention,
- distinguish between watchlist monitoring and active remediation,
- manage document backlog more effectively,
- improve visibility over ageing items and closure progress,
- support stronger audit readiness. :contentReference[oaicite:7]{index=7}

## Recommended Actions
1. **Triage immediately** — prioritise PEP-linked, failed-verification, and ageing files.  
2. **Clear the backlog** — work remaining remediation cases in controlled batches with clear ownership and due dates.  
3. **Tighten controls** — improve intake checks, verification rules, and document expiry monitoring.  
4. **Report weekly** — track ageing, overdue items, PEP files, failed verification, and closure rates. :contentReference[oaicite:8]{index=8}
